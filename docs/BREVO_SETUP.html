Brevo Email Digest Setup Guide

This document walks you through the one-time setup needed to enable the IBD LitMonitor email digest system.

Overview

The system works like this:

  1. You maintain a Google Sheet with subscriber email addresses
  2. The sheet is published as a CSV (read-only, URL kept private as a GitHub secret)
  3. After each daily render, GitHub Actions runs scripts/send_digest.py
  4. The script reads the subscriber list and sends a digest email via Brevo (free up to 300 emails/day)

Step 1 — Create a Brevo account

  1. Go to brevo.com and sign up for a free account
  2. Verify your sender email address: ibdlitmonitor@gmail.com
    • In Brevo: Senders & IPs → Senders → Add a sender
    • Name: IBD LitMonitor, Email: ibdlitmonitor@gmail.com
    • Verify via the confirmation email Brevo sends
  3. Generate an API key:
    • Go to Account settings → API Keys → Generate a new API key
    • Name it github-actions
    • Copy the key — you’ll add it to GitHub secrets next

Step 2 — Add GitHub secrets

In your private repo (ibd-litmonitor-src): - Settings → Secrets and variables → Actions → New repository secret

Add these two secrets:

Secret name Value
BREVO_API_KEY Your Brevo API key from Step 1
SUBSCRIBER_SHEET_URL The Google Sheet CSV export URL (see Step 3)

Step 3 — Create the subscriber Google Sheet

  1. Create a new Google Sheet at sheets.google.com

  2. Name it IBD LitMonitor Subscribers

  3. Add these exact column headers in row 1:

    A B C D
    email name frequency active

  4. Column notes:

    • email — subscriber email address (required)
    • name — first name for personalized greeting (optional, leave blank if unknown)
    • frequency — daily or weekly (defaults to daily if blank)
    • active — TRUE to receive emails, FALSE to pause (defaults to TRUE)

  5. Add yourself as a test subscriber in row 2:

    dahhamalsoud@gmail.com  |  Dahham  |  daily  |  TRUE

  6. Publish the sheet as CSV:

    • File → Share → Publish to web
    • Select Sheet 1 and Comma-separated values (.csv)
    • Click Publish and copy the URL
    • The URL looks like: https://docs.google.com/spreadsheets/d/SHEET_ID/export?format=csv&gid=0

  7. Add the CSV URL as the SUBSCRIBER_SHEET_URL GitHub secret (Step 2)

Step 4 — Create the subscribe Google Form (optional)

To let users self-subscribe without emailing you:

  1. Go to forms.google.com → Create new form
  2. Title: Subscribe to IBD LitMonitor
  3. Add questions:
    • Email address (short answer, required, validate as email)
    • First name (short answer, optional)
    • Frequency (multiple choice: Daily / Weekly)
  4. Connect to your subscriber sheet:
    • Responses tab → Link to Sheets → Select your subscriber sheet
  5. Set up a Google Sheets App Script (or use Zapier/Make) to automatically:
    • Add active = TRUE to new rows
    • Normalize frequency to daily/weekly
  6. Get the embed code:
    • Send → <> (embed) → Copy the iframe HTML
  7. Paste the iframe code into subscribe.qmd replacing the placeholder comment

Step 5 — Test the digest

Run a dry-run locally to verify the script works:

export SUBSCRIBER_SHEET_URL="your-csv-url"
python3 scripts/send_digest.py --dry-run

To send a real test email:

export BREVO_API_KEY="your-key"
export SUBSCRIBER_SHEET_URL="your-csv-url"
python3 scripts/send_digest.py

Or trigger from GitHub Actions manually (Actions tab → Update IBD LitMonitor → Run workflow).

Free tier limits

  • Brevo free: 300 emails/day, unlimited contacts
  • With ~100 daily subscribers and ~50 weekly: well within free tier
  • If you grow beyond 300/day, Brevo Starter is $9/month for 5,000/day

Troubleshooting

Problem Fix
SUBSCRIBER_SHEET_URL not set Add the secret to GitHub repo settings
BREVO_API_KEY not set Add the secret to GitHub repo settings
HTTP 401 from Brevo API key is wrong or expired — regenerate in Brevo
HTTP 400 from Brevo Sender email not verified in Brevo — verify ibdlitmonitor@gmail.com
Emails going to spam Set up SPF/DKIM in Brevo for better deliverability
Sheet fetch fails Make sure the sheet is published to web as CSV
